# Criterios de Evaluación

## Elementos a evaluar

1. **Funcionalidad básica**  
   - La aplicación se ejecuta sin errores.  
   - Muestra la hora en pantalla.  
   - La hora se actualiza automáticamente cada segundo.  

2. **Interactividad**  
   - Se incluye un botón en la interfaz.  
   - Al pulsar el botón, cambia el diseño del reloj (tamaño, color, borde, fondo, etc.).  
   - El cambio de diseño es cíclico y recorre al menos tres estilos diferentes.

3. **Organización y limpieza del código**  
   - Uso adecuado de variables de estado (`remember` y `mutableStateOf`).  
   - Empleo correcto de `LaunchedEffect` y `delay` para la actualización.  
   - Código indentado y comentado donde sea necesario.  

4. **Creatividad y mejoras opcionales** (bonus)  
   - Se agrega un título y/o fecha además de la hora.  
   - Se implementa un cuarto estilo propio.  
   - Se modifica el color de fondo en cada estilo.
